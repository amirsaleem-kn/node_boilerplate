Logger #

Path: ./server/lib/logger/Logger.js
  • Dependencies
    • ChalkJs

The Logger class provides methods to print logs to terminal and stores log files in the directory. 

// import logger class
const Logger = require('logger/Logger'); // relative path needs to be specified

let logger = new Logger(); // create an instance of the class

// log some output 
logger.log('cool');

Singleton pattern can also be achieved by using the following code 

const logger = require('logger'); // creates an instance and exports the same

logger.log('some stuff'); // same instance can be used throughout the application

The logging methods also stores logs in files when application is not running in production environment. Environment is specified by using NODE_ENV variable. The log files are saved by date. The default path for log files is /etc/logs/ but can be changed by passing the options in constructor function. Logger class can be instantiated by using the following options 

const Logger = require('logger/Logger');

const logOptions = {
    writeToFile: true // Boolean (true | false) to specify weather logs should be saved to file or not
    filePath: '/my/awesome/folder' // a directory to specify where to save log files
}

logger = new Logger(logOptions) // create an instance with specified options

Logger.log(args) #

The Logger.log(args) calls console.log(args) if environment is not production. Useful for general purpose stdout logging while debugging or looking for a specific output of the program. Does not prints anything in production envrionment.

// import logger
const logger = require('logger');

// print something on the console
logger.log('some cool stuff');

// you can pass multiple arguments
logger.log('some cool stuff', 'some really cool stuff');

Logger.warning(args) #

This method works only in test. The Logger.warning(args) method prints orange colored statements. If argument is string, [warning] is prefixed 

// import logger
const logger = require('logger');

// print something on the console
logger.warning('The call_stack() function has been deprecated, consider using call_queue() instead');
// output: [warning]: The call_stack() function has been deprecated, consider using call_queue() instead

// you can pass multiple arguments
logger.warning('some warning', 'some other warning');
